# T-02 教材草稿模板

> **模板编号**：T-02
> **文件角色**：📘 教材备课底稿（按课次合并 5 模块）
> **所属阶段**：备课阶段（位于 T-01 之后，正式课件之前）
> **配套输入**：T-01 知识点草稿模板.md（按知识点准备好的备课单元）
> **配套下游**：《第 X 课：标题（知识点）》正式课件，存入 `BOOK/NN 章节名/` 目录

> **使用场景**：把 `T-01 知识点草稿` 里按"5 步"准备好的每个知识单元，组装成 **2 小时完整课**。

> **适用课型**：📘 知识讲解课 / 🐢 海龟绘图课（**不**适用于 T-03 考级课、T-04 章节复习课）。

## 🎨 跨节通用代码规范

> 下面 7 条规则**贯穿 x.0~x.4 所有节的代码块**，请严格遵守。

1. **可直接运行**：禁止伪代码、`<xxx>` 占位符、`…` 省略号（除"省略写法"教学外）。
2. **必须带 `#` 行内注释**：每行代码后都要说"这行在做什么"。
3. **示例递进**（仅对 x.0 第一次热身、x.1 第一次运行强制）：
   - 示例 1：**最小化**，3~5 行，只展示"骨架"。
   - 示例 2：**贴近情境**，5~10 行，用 Minetest 场景包装。
   - 示例 3（可选）：**联合使用**，覆盖本节**全部新知识点**。
4. **错误代码必须能复现报错**：在 Bug 猎人里写"能跑出报错"的代码，而不是凭空虚构。
5. **修复代码必须带注释说明修复点**：用 `# 修复: ...` 注释。
6. **任务代码给"参考实现"而非"标准答案"**：学生可能有不同写法。
7. **跨节主题一致**：5 节必须**沿用同一个 Minetest 主题**（如本课是"魔法背包"则 5 节都围绕它），不要中途换地图。

## x.1 章节引入与知识点介绍

> **⏱ 时间预算**：25~30 分钟
> - 背景介绍 3~5 分钟
> - Multi-TRIX百科 10~12 分钟
> - 第一次运行 12~15 分钟

### 背景介绍

搭建一个知识学习场景，帮助课堂引入知识点，激发学生的兴趣。

**写作要求**：
- **视角**：以"Multi-Boy 冒险者"或"玩家"作为主语（不要用"我们"这种含糊视角）。
- **长度**：1~3 句话，不超过 80 字（比 011 模板宽一些，因为是 2 小时课的主情境）。
- **钩子**：最后一句要**指向本节核心动作**。
- **跨节呼应**：5.4 的"通关仪式"要呼应本节的主人公/事件。

### Multi-TRIX百科

介绍本节课的核心知识点，使用生动形象的比喻来帮助学生理解。
每个知识点都要有一个简短的定义和一个使用方法的示例。

**格式（必填）**：
```markdown
- **知识点名称** = 比喻：简短定义。
  > 使用方法：示例和说明。
```

**写作要求**：
- **比喻必须具体**：每个知识点一个**生动具体**的物品/角色比喻（不要用"工具"、"概念"这种空泛词）。
  - 优秀比喻举例：`print()` = 喊话器；变量 = 背包；注释 = 便利贴；`import` = 打开工具包；`Turtle()` = 召唤助手；`forward(100)` = 下达命令；列表 = 魔法背包；切片 = 抽屉切片刀；`len()` = 物品计数器。
- **一个知识点一条**：不要把多个知识点混在同一条 bullet 里。
- **定义 ≤ 30 字**：一句话能说清。

### 第一次运行

通过多个简单的代码示例，展示本节课的核心知识点，让学生第一次体验编程的乐趣。
每个示例都要有注释来解释代码的作用。
示例包含全部知识点的使用，帮助学生建立知识点之间的联系。

**写作要求**：
- **递进式 3 步**（遵循上文"跨节通用代码规范"第 3 条）：
  1. **示例 1（最小化）**：3~5 行，只展示本节"骨架"。
  2. **示例 2（贴近情境）**：5~10 行，用 Minetest 场景包装。
  3. **示例 3（联合使用，可选）**：覆盖本节**全部新知识点**，是 x.3 创造任务的"原型"。
- **不要重复 5.0**：5.0 复习过的语法，5.1 第一次运行**不要再写一遍最小化示例**。

## x.2 Bug猎人 老师报错了！你能帮老师修复代码吗？

> **⏱ 时间预算**：25~30 分钟
> - Bug 猎人 15~20 分钟
> - 代码修复家 10 分钟

### 常见错误类型总结

> 在列出具体 Bug 之前，先在 x.2 顶部用 1~2 行/表总结本节涉及的 3~5 种错误类型。
>
> 错误类型库：
> - `IndexError`：索引/切片越界
> - `ValueError`：`remove()` 删不存在的值；切片步长为 0
> - `TypeError`：对不支持的类型用切片/索引
> - `SyntaxError`：语法错误（缺逗号、漏括号）
> - `AttributeError`：方法名拼写错误（如 `.apend()`）

### Bug 猎人：4~5 个 Bug

> 给出**有 Bug 的代码 → 报错信息 → 你的修复**三段式。

**写作要求**：
- **每个 Bug 一段**：标题 `### Bug 1: 简明名称`。
- **报错必须真实可复现**：先在本地跑一遍，确认能跑出预期的报错。
- **修复代码必须带 `# 修复: ...` 注释**。
- **每节课的 4~5 个 Bug 覆盖至少 3 种错误类型**（避免都报 `IndexError`）。
- **错误场景尽量复用 5.1 第一次运行会出现的真实代码**——学生能认出"哦这就是我刚才写的"。

### 代码修复家：1~2 个综合任务

> 给出**更复杂、更长、含 2~3 个 Bug**的代码片段，让学生综合修复。

**与 Bug 猎人的边界**：
- **Bug 猎人**：每个 Bug 是**独立的 3~5 行**代码块，**单一**错误，5 分钟内能修一个。
- **代码修复家**：1 段**完整 8~15 行**的代码，**多 Bug 复合**（2~3 个错），需要学生"读懂意图"后才能修对。

**写作要求**：
- **任务有明确目标输出**（如"修复后应输出 `[A, B, C]`"），避免"修好就行"这种模糊目标。
- **提供"有 Bug 的代码 → 你的修复"两段对照**。
- **至少 1 个**代码修复家任务，作为本节的"高难度挑战"。

## x.3 创造与协作

> **⏱ 时间预算**：30~40 分钟
> - 基础任务 2 个，共 15~20 分钟
> - 进阶任务 2 个，共 15~20 分钟

根据本节课的知识点，设计一些创造性的编程任务，让学生通过编程来实现自己的创意。
每个任务都要有明确的目标和要求，帮助学生理解任务的内容和目标。
通过完成任务，帮助学生提升编程能力和创造力。

可以结合本节课的背景故事，设计一些与 Minetest 相关的编程任务，帮助学生将所学知识应用到实际情境中。

任务可以包括基础任务和进阶任务，帮助学生逐步提升编程能力。

### 任务设计原则

| 维度 | 基础任务 | 进阶任务 |
| :--- | :--- | :--- |
| **知识点数量** | 1~2 个（聚焦本节新内容） | 3+ 个（跨节综合 / 跨章综合） |
| **代码行数** | 3~8 行 | 10~20 行 |
| **思维能力** | 模仿 / 替换 | 设计 / 拆分 / 调试 |
| **目标输出** | 必须明确（学生知道什么算"完成"） | 可模糊（鼓励多种解法） |
| **完成率预期** | ≥ 80% 学生能完成 | 50% 学生能完成 |

### 基础任务（≥ 2 个）

> 直接套用"### 任务 N: 标题"的格式。

**写作要求**：
- **任务标题具体到能直接动手**（如"战利品数量统计员"，而不是"练习题 1"）。
- **任务描述用 bullet 列出步骤**（如"① 创建列表 ② 用 len 打印 ③ 切片打印前 3 件"），便于学生照着做。
- **教师给"参考实现"**：用 markdown 代码块包裹，**折叠**起来（用 `>` 引用）以免学生直接抄。

### 进阶任务（≥ 2 个）

> 写法同基础任务，但**允许使用 x.0 复习的旧知识**或**涉及 x.3 之前章节的内容**。

**写作要求**：
- **明确"挑战"二字**：标题加"（挑战）"或"进阶"标识。
- **题目要有一定开放性**：可设 2~3 个小目标，让学生挑。
- **必须呼应 x.1 的 Minetest 主线情境**。

## x.4 复盘与固化

> **⏱ 时间预算**：15~20 分钟
> - 速记挑战 3~5 分钟
> - 我的冒险笔记 8~10 分钟
> - 通关仪式 3~5 分钟

根据本节课的内容，设计一些复盘和固化的活动，帮助学生巩固所学知识。
每个活动都要有明确的目标和要求，帮助学生理解活动的内容和目标。
通过完成活动，帮助学生巩固所学知识并提升编程能力。

### 速记挑战

给出一些编程相关的术语或概念，让学生在规定时间内（比如 1 分钟）记住并写下来，帮助学生巩固知识点。

**写作要求**：
- **3~5 个术语**：必须是本节**新学的、关键的**术语。
- **1 分钟计时**：让学生有紧迫感。
- **不超纲**：不要把后面章节的术语混进来。

### 我的冒险笔记

简洁的文字、口诀或表格，让学生记录自己的学习心得和体会，帮助学生反思和总结所学内容。

**写作要求**：
- **必含表格或口诀**：不允许写大段散文。
- **80~150 字内可抄完**：5 分钟内能完成。
- **覆盖本节全部新知识点**：每个新语法 1 行表格 / 1 句口诀。
- **可选："💡 红石小技巧"或"⚠️ 易错点"** 提醒 1~2 条。

### 通关仪式

> 模板默认是空的，需要你**必须**填写。**不要**让"通关仪式"留空。

**格式（必填）**：
```markdown
### 通关仪式

🎉 恭喜你！你已经成功掌握了 [本节核心知识点的 1 句话总结]。

你现在可以：
- ✅ [本节最基础的能力 1]
- ✅ [本节中等能力 2]
- ✅ [本节进阶能力 3]
- ✅ [本节迁移能力 4]（可选）

带上你新学到的 [本节核心比喻]，继续在 Minetest 和编程的世界里冒险吧！下次，我们将学习 [下节课预告]。冒险家，下节课见！🚀
```

**写作要求**：
- **3~5 条"✅ 你已经能…"**：覆盖本节核心能力，由易到难。
- **呼应 x.1 背景介绍**：用相同的主人公/事件做收束。
- **预告下节课**：用 1 句话。

### 角色分工

- **011** = "按知识点拆开写" —— 一节课可能产生 3~5 个独立的"知识点单元"，其中可能包含 1 个**复习单元**。
- **012** = "按课次统一组装" —— 把 011 的所有单元合并：
  - 复习单元 → 折进 **x.0 上节回顾**（如有）。
  - 新知识点单元 → 折进 **x.1** 的 Multi-TRIX百科 + 第一次运行。
  - 之后补充 x.2 / x.3 / x.4，构成完整 2 小时课程。
- **T-03 考级练习课** 和 **T-04 章节复习课** 是另外两个独立模板，本模板不涉及。
