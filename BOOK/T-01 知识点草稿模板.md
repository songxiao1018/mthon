## ✏️ 知识点 0001：【知识点名称】

### 一、Minetest 情境引入

> **目的**：用 Minetest 游戏世界作为背景，搭建一个"为什么要学这个"的故事场景。
> 把抽象的语法点映射到玩家在游戏里**真正会遇到**的具体情境。

**写作要求**：
- **视角**：以"Multi-Boy 冒险者"或"玩家"作为主语（不要用"我们"这种含糊视角）。
- **长度**：1～3 句话，不超过 60 字。
- **目的**：让学生产生"我玩游戏时确实遇到过这个问题"的代入感。
- **钩子**：最后一句要**指向本知识点的核心动作**（如"你需要学会喊话"、"你需要记录宝藏数量"）。

**样例参考**：
- ✅ "你出生在 Minetest 的大草原，身上只有 3 个钻石。你按 `T` 键说话，但系统听不懂中文。你必须学会 Python 语言，才能告诉系统你的财富！"
- ✅ "你在 Minetest 的海滩上发现了一只老海龟，它说：'只有懂得指挥之术的人，才能让我帮你画图。'"
- ✅ "你在村庄里发现了一堆干草块，想把它们收集并记录下来，你该用什么样的变量来存储数量？"
- ❌ "今天我们学习 print 函数。" （没有情境，钩子太弱）

### 二、核心知识点

> **目的**：用 **"比喻 + 定义 + 使用方法"** 三段式，定义本知识点的核心概念。

**格式（必填）**：
```
- **【知识点名称】 = 【比喻】**：【一句话简短定义】。
  > 使用方法：【最小可运行示例】
  > 示例：【贴近情境的扩展例子】
```

**写作要求**：
- **比喻优先**：每个知识点都要有一个**生动具体**的比喻（不要用"工具"、"概念"这种空泛词）。
  - 优秀比喻举例：`print()` = 喊话器；变量 = 背包；注释 = 便利贴；`import` = 打开工具包；`Turtle()` = 召唤助手；`forward(100)` = 下达命令。
- **定义要短**：一句话能说清，不要超过 30 字。
- **使用方法要可粘贴**：给出**最简形式**的代码（如 `变量名 = 值`）。
- **示例要贴近情境**：用游戏中的物品/场景作例子（如 `diamonds = 3`，不用 `x = 3`）。

**样例参考**：
- ✅ "`print()` = 喊话器：可以让你在屏幕上显示任何你想说的话。
  > 使用方法：`print(内容)`
  > 示例：`print("Hello, World!")` 就是对着屏幕喊一句话。"
- ✅ "**导入库 (import)** = 打开工具包：从 Python 的'百宝箱'里取出一个包含特定功能的工具包。
  > 使用方法：`import turtle` 或 `import turtle as t`
  > 示例：`import turtle` 意思是'我要使用画图的工具包'。"
- ✅ "**整数变量 (Integer Variable)** = 数字容器：一个用来存储整数的'盒子'。
  > 使用方法：`变量名 = 整数值`，例如 `score = 10`
  > 示例：`score = 10` 意思是'我创建了一个叫 score 的变量，并把数字 10 放进去'。"
- ❌ "print 是 Python 的输出函数。" （没有比喻，没有示例）

**填写区**：

### 三、代码演示（第一次运行）

> **目的**：给一个**可直接运行**的最小代码示例，让学生**第一次看到**这个知识点的样子。

**写作要求**：
- 代码必须**可以直接复制运行**，不要有伪代码。
- 每个示例都要有**行内注释**（用 `#`），说明每行代码的作用。
- 一个知识点**配 1～3 个示例**，示例之间要有**递进关系**：
  - 第一个示例：**最小化**，只展示这个知识点的"骨架"（3～5 行）。
  - 第二个示例：**贴近情境**，用游戏场景包装（5～10 行）。
  - 第三个示例（可选）：**变化形式**，展示这个知识点的"另一种用法"。
- 示例要能**覆盖本节课全部知识点的联合使用**，帮助学生建立知识点之间的联系。

**样例参考（输出与变量课，3 个示例递进）**：
```python
# 示例 1：最小化——只输出字符串
print("Hello, World!")

# 示例 2：贴近情境——把钻石数量存到变量里
diamonds = 3
print(diamonds)

# 示例 3：联合使用——变量 + 输出 + 注释
# 这是 Multi-Boy 的财富清单
gold = 100
diamond = 3
print("我的金币：", gold)
print("我的钻石：", diamond)
```

### 四、互动提问

> **目的**：在讲完知识点后，**立即**抛 1～2 个由浅入深的小问题，让学生**马上动手**敲一遍。
> 这是"听懂了"到"会用了"的关键过渡。

**格式**：
- **必问问题（基础）**：用本知识点解决一个**最简单**的场景。
- **选问问题（拓展）**：在简单场景上做**一点变化**（如改数字、换名字、加要求）。

**写作要求**：
- 问题描述要**具体到能直接敲代码**（不要问"什么是变量"这种开放题）。
- 问题要**与情境引入呼应**（用游戏中的物品/角色作例子）。
- 一节课的多个问题之间要**有梯度**（从改一个数字 → 加一个新变量 → 改一种用法）。
- 在填写区下方，**给老师预留"参考解答"**（用 `>` 引用块折叠）。

**样例参考**：
- ✅ 必问："请问如何定义一个变量来存储你的年龄，并输出它？"
  > 参考：`age = 25`  `print(age)`
- ✅ 必问："请用 `forward()` 和 `left()` 画一个边长为 100 的正方形。"
- ✅ 选问："如果让你在输出时把钻石数量加 1 再打印，该怎么改？"
  > 参考：`diamonds = diamonds + 1`  `print(diamonds)`

### 五、随堂笔记

> **目的**：用 1～3 段**简洁、易抄写**的文字，让学生把本知识点记在笔记本上。
> 这一栏的内容**直接决定学生考试时能不能回忆起来**。

**写作要求**：
- 语言要**学生能看懂**，避免堆砌"对象"、"实例"、"类型"等高深术语。
- 优先用**口诀、顺口溜、表格、问答**等可视化形式，不要写大段散文。
- 控制在 **80～150 字**，5 分钟内能抄完。
- 可选：加一个 **"💡 红石小技巧"** 或 **"⚠️ 易错点"** 提醒。

**常见形式**：
- **口诀式**："召唤三步曲：一导库(`import`)，二造龟(`Turtle()`)，三下令(`.方法()`)。"
- **表格式**：

  | 知识点 | 解释 | 例子 |
  | :--- | :--- | :--- |
  | `print()` | 用来在屏幕上显示内容的函数 | `print("Hello")` |
  | 变量 | 用来存储数据的容器，可以随时修改 | `diamonds = 3` |
  | 注释 | 用来解释代码的文字，不会被执行 | `# 这是注释` |

- **问答式**："Q：变量名能不能以数字开头？A：不能，会报 `SyntaxError`。"
