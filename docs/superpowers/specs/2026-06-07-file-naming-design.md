## 一、背景与动机

### 1.1 项目现状

`BOOK/` 目录现承担三件事：

1. **模板文件**（备课用）：`000 章节安排.md`、`011 知识点草稿模板.md`、`012 教材草稿.md`、`013 考级练习课.md`、`014 章节复习课.md`、`015 Slidev课件设计 脚本安排设计.md`、`016 Slidev课件设计 演示文件设计.md`
2. **正式课件**（课堂用）：`BOOK/NN 章名/NNN 第 X 课：标题（知识点）.md`
3. **草稿 / 物料**：`BOOK/NN 章名/草稿 *...md`、`BOOK/草稿 课件/...`

### 1.2 当前 6 类命名"病状"

| 病状 | 典型反例 |
| :--- | :--- |
| 模板编号断层 / 含义模糊 | `000`、`011`、`012`、`013`、`014`、`015`、`016`——`015/016` 与 `013/014` 没有任何语义关联，读者无法从编号推断模板职责 |
| 草稿命名风格混乱 | 同目录并存 `草稿 001_第2课_海龟世界_教学内容.md`（下划线）和 `草稿 004 第 十一 章 第 1 节 认识列表.md`（空格 + 中文数字） |
| 正式课件"编号 vs 课次"重复 | `005 第 5 课：列表操作（截取与长度）.md`——"005"和"第 5 课"信息冗余 |
| 章节文件夹命名不一致 | `01 输出与变量` vs 表格中 `12 综合案例 1`——编号 OK 但"章"概念模糊 |
| 课件附件无课次 | `草稿 课件/Python 列表操作（截取与长度）互动课件大纲设计方案.docx` 没有 `00X_` 前缀 |
| `.gitignore` 规则不严密 | `草稿*` 只匹配文件，但 `BOOK/草稿 课件/` 目录里的子文件因不带"草稿"前缀仍能入库 |

### 1.3 修订边界

经用户确认：本次**全部纳入**——覆盖根级模板、章节文件夹、正式课件、草稿、互动课件附件、`.gitignore` 规则、Slidev 目录。

## 二、全局 4 条原则

贯穿全部规则的 4 个硬性约定：

1. **零填充固定位数**
   - 课次：`001` ~ `051`（3 位）
   - 章节：`01` ~ `15`（2 位）
   - 模板：`T-00` ~ `T-99`（2 位）
2. **空格分隔符**：所有 token 之间用 **1 个半角空格**（不用下划线、不用中文标点、不用连字符 `T-XX-角色`）
3. **层级目录而非长文件名**：用文件夹表达"是什么"（如 `课件物料/旧版互动课件/`），文件名只承载"是哪个"
4. **`草稿 ` 前缀**触发 `.gitignore` 排除；正式交付物不带此前缀

## 三、根级模板文件规则（`T-XX` 语义 ID）

### 3.1 编号体系

| 编号 | 名称 | 角色 | 替换旧文件 |
| :--- | :--- | :--- | :--- |
| `T-00` | 章节安排 | 全书进度总表 / 课次与章映射的唯一真源 | `000 章节安排.md` |
| `T-01` | 知识点草稿模板 | 按知识点拆开写（5 步单元） | `011 知识点草稿模板.md` |
| `T-02` | 教材草稿模板 | 按课次合并 5 模块（x.0~x.4） | `012 教材草稿.md` |
| `T-03` | 考级练习课模板 | 13 节 GESP 真题刷题 | `013 考级练习课.md` |
| `T-04` | 章节复习课模板 | 章末综合大项目 | `014 章节复习课.md` |
| `T-05` | Slidev 脚本安排 | Slidev 课件逐页脚本 | `015 Slidev课件设计 脚本安排设计.md` |
| `T-06` | Slidev 演示文件 | Slidev 可直接渲染的 `slides.md` | `016 Slidev课件设计 演示文件设计.md` |
| `T-07` | 知识点总表 | 全书 `### 知识点 NNNN 名称` 编号表 | `草稿 课件/知识点.md`（上提） |

### 3.2 命名格式

**正则**：`T-\d{2} + 半角空格 + 中文模板名.md`

- 编号 2 位零填充（`T-00` ~ `T-99`），预留到 `T-99` 不用换体系
- 文件名内**不**写"模板"二字（如 `T-01 知识点草稿模板.md` 即可，含义自明）

### 3.3 模板内部统一头部（元信息块）

每个 `T-XX` 文件 H1 标题正下方**必须**包含的 5 行元信息块。各模板的"文件角色"标准短语如下表：

| 模板 | 文件角色标准短语 |
| :--- | :--- |
| T-00 | 🗂 章节安排总表（全书进度真源） |
| T-01 | 📘 知识点备课底稿（按知识点粒度） |
| T-02 | 📘 教材备课底稿（按课次合并 5 模块） |
| T-03 | 📝 考级练习课底稿（13 节 GESP 真题刷题） |
| T-04 | 🔄 章节复习课底稿（章末综合大项目） |
| T-05 | 🎬 Slidev 脚本安排（逐页脚本表） |
| T-06 | 🎬 Slidev 演示文件（可直接渲染的 slides.md） |
| T-07 | 📚 知识点总表（`### 知识点 NNNN 名称` 编号真源） |

以 T-01 为例，标准元信息块格式如下：

```markdown
# T-01 知识点草稿模板

> **模板编号**：T-01
> **文件角色**：📘 知识点备课底稿（按知识点粒度）
> **所属阶段**：备课阶段（位于 T-02 教材草稿**之前**）
> **配套输入**：T-00 章节安排.md（本课涉及的知识点编号、所属章节、课时类型）
> **配套下游**：T-02 教材草稿模板.md
```

- 5 行元信息为**强约束**：模板编号、文件角色、所属阶段、配套输入、配套下游
- "文件角色"必须使用上表的标准短语，不得自创

### 3.4 跨模板引用约定

- 模板内部互相引用一律写完整文件名：`T-02 教材草稿模板.md`
- 模板与课件互相引用：模板侧写"配套下游 005 列表操作（截取与长度）.md"；课件侧写"上游 T-02 教材草稿模板.md"

## 四、章节文件夹与正式课件规则

### 4.1 章节文件夹

**正则**：`\d{2} + 半角空格 + 章节名`

- 编号对齐 `T-00 章节安排.md` 的"章节内容"列（2 位零填充）
- 章节名 = 表格中的章节名（不带"第 X 章"前缀）
- 现有 `01 输出与变量/`、`02 输入与数据类型/`、`11 列表与元组/` 全部**符合**此规则，**无需重命名**

### 4.2 正式课件文件名

**正则**：`\d{3} + 半角空格 + 短标题.md`

- 3 位零填充，对齐 `T-00 章节安排.md` 中的"总课次"列（`001`~`051`）
- **去掉**"第 X 课："前缀（"005"已表达课次，与"第 5 课"信息冗余）
- **短标题**：取自 H1 标题的"括号外"部分；如 `第 1 课：向世界问好（输出与变量）` → 短标题 `向世界问好（输出与变量）`

### 4.3 正式课件内部 H1 标题

H1 标题**保留**完整形式（含"第 X 课"），便于人读和打印：

```markdown
# 001 向世界问好（输出与变量）

> **课次**：第 1 课
> **所属章节**：第 1 章 输出与变量
> **课时类型**：📘 知识讲解课
> **本课涉及的知识点编号**：0001、0002
> **上游模板**：T-02 教材草稿模板.md
```

### 4.4 课次编号的唯一真源

`T-00 章节安排.md` 是**唯一**确定 `001` ~ `051` 对应哪节课的地方。任何新增课必须先改 `T-00` 再创建文件。

## 五、草稿文件规则

### 5.1 草稿文件命名格式

**正则**：`草稿 + 半角空格 + T-\d{2} + 半角空格 + \d{3} + 半角空格 + 短标题.md`

- `草稿 ` 前缀保证被 `.gitignore` 的 `草稿*` 规则排除
- `T-XX` 标记**这个草稿用哪个模板写**（同一节课可有 T-01/T-02/T-03/T-04 多份草稿）
- `\d{3}` 对齐正式课件的课次号

### 5.2 典型场景示例

| 场景 | 文件名 |
| :--- | :--- |
| 第 1 课的"知识点草稿"（来自 T-01） | `草稿 T-01 001 向世界问好.md` |
| 第 1 课的"教材草稿"（来自 T-02） | `草稿 T-02 001 向世界问好.md` |
| 第 5 课的"考级练习课草稿" | `草稿 T-03 005 列表操作.md` |
| 第 5 课的"Slidev 脚本草稿" | `草稿 T-05 005 列表操作.md` |
| 第 5 课的"Slidev 演示草稿" | `草稿 T-06 005 列表操作.md` |

### 5.3 草稿与正式课件的对应关系

- 草稿文件**必须**与对应的正式课件**放在同一目录**（章节文件夹下），方便 git diff 时一起查看
- 草稿 H1 标题**必须**标注"【草稿】"前缀，避免误当成正式课件打开：

```markdown
# 【草稿】T-01 → 001 向世界问好（输出与变量）

> **对应正式课件**：`001 向世界问好（输出与变量）.md`
> **配套上游**：`T-00 章节安排.md`
```

### 5.4 草稿删除与归档

- 正式课件交付（git commit）后，**草稿可保留**，但**不允许**就地修改
- 草稿的归档由 `.gitignore` 保障，**永远不进版本控制**

## 八、跨文件引用约定

### 8.1 引用格式

任何 markdown 内的文件引用，**统一**使用相对路径 + 完整新文件名：

```markdown
参见 [T-02 教材草稿模板.md](T-02 教材草稿模板.md)
本节课对应草稿：[草稿 T-01 001 向世界问好.md](草稿 T-01 001 向世界问好.md)
上游进度表：[T-00 章节安排.md](../T-00 章节安排.md)
```

### 8.2 引用场景速查

| 引用方 | 被引用方 | 写法 |
| :--- | :--- | :--- |
| 模板 ↔ 模板 | `T-XX` | `T-02 教材草稿模板.md` |
| 模板 → 课件 | `NNN 标题.md` | `005 列表操作（截取与长度）.md` |
| 课件 → 模板 | `T-XX` | `T-02 教材草稿模板.md` |
| 课件 → 草稿 | `草稿 T-XX NNN 标题.md` | `草稿 T-01 001 向世界问好.md` |
| 课件 → 课件 | `NNN 标题.md` | `005 列表操作（截取与长度）.md` |
| 课件 → 知识点 | `T-07 知识点总表.md` + 编号 | `T-07 知识点总表.md`（锚点 `0015`） |
| 课件 → 物料 | `课件物料/...` | `课件物料/005 列表操作 课件.pdf` |

### 8.3 知识点的"全局编号"引用

`T-07 知识点总表.md` 是 `### 知识点 NNNN 名称` 编号表的唯一真源。任何课件引用某个知识点时：

- **写作时**：写"知识点 0001"或"0001 输出字符串"
- **链接到 T-07**：用锚点 `[0001 输出字符串](../T-07 知识点总表.md#0001-输出字符串)`

## 九、重构迁移计划

### 9.1 迁移原则

- **不丢内容**：所有旧文件**必须**有去处（改名 / 移入子目录 / 上提为模板）
- **git 历史可追溯**：用 `git mv` 改名（**不要**用文件系统重命名后 `git add`，否则丢失 rename detection）
- **分批提交**：每个 Phase 单独 commit，便于回滚

### 9.2 迁移步骤（7 个 Phase）

| Phase | 动作 | 涉及文件数 | 风险 |
| :---: | :--- | :---: | :---: |
| **P1** | 更新 `.gitignore`：删除 `CLAUDE.md` 规则；新增 `BOOK/课件物料/草稿*` 等防御规则 | 1 | 低 |
| **P2** | 重命名 BOOK/ 根级模板：`000` → `T-00`，`011~016` → `T-01~T-06` | 7 | 低 |
| **P3** | 上提 `草稿 课件/知识点.md` → `T-07 知识点总表.md`；删除 `草稿 课件/草稿.md` | 2 | 低 |
| **P4** | 在 `T-00 ~ T-07` 每个模板的 H1 下方**插入** § 3.3 规定的 5 行元信息块 | 8 | 中 |
| **P5** | 重命名正式课件：去掉"第 X 课："前缀，保留 3 位顺序号与短标题 | 5 | 低 |
| **P6** | 重命名草稿：统一为 `草稿 T-XX NNN 短标题.md` | 6 ~ 9 | 中 |
| **P7** | 重构 `草稿 课件/` → `课件物料/`，子目录归类、旧版归档改名 | 8 | 中 |
| **P8** | 全项目 grep 旧文件名（`011 知识点`、`第 1 课：`、`草稿 课件` 等），更新内部 markdown 链接 | — | 高（散落引用） |

## 十、待决风险与回滚预案

### 10.1 已知风险

1. **git rename detection 失效**：若用文件系统重命名（Explorer / Finder），git 会视为"删除 + 新增"，丢失 blame 记录。**必须用 `git mv`**。
2. **P8 内部链接遗漏**：模板 / 课件 / 草稿互相引用若忘记更新，会出现 404。需要在每个 Phase 后跑 `grep -r "011 知识点\|012 教材\|013 考级\|014 章节\|015 Slidev\|016 Slidev\|草稿 课件\|第 1 课：\|第 2 课：\|第 3 课：\|第 4 课：\|第 5 课：" BOOK/ --include="*.md"`。
3. **CLAUDE.md 误删指令**：当前 `CLAUDE.md` 误被 `.gitignore` 排除。P1 删除该规则后，需要 `git add -f CLAUDE.md` 强制入库。
4. **草稿归类推断可能错**：P6 推断的 6 项草稿映射需逐一打开校验，特别是"海龟编程"是否真的属于 T-02（教材草稿）。

### 10.2 回滚预案

- 每个 Phase 独立 commit；任何 Phase 失败可 `git reset --hard HEAD~1` 回滚到上一稳定点
- 推荐使用 `git worktree` 在隔离分支中执行，验证通过再 merge
